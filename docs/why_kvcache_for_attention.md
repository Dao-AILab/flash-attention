# 为什么新生成的token与历史token进行注意力计算时需要使用KV缓存

## 1. Transformer自回归生成的本质

Transformer模型在进行文本生成等自回归任务时，遵循以下基本原则：

- **时序依赖性**：每个新生成的token依赖于之前所有生成的token
- **因果关系约束**：当前token只能「看到」之前的token，不能访问未来的token
- **上下文完整性**：为了生成连贯的内容，模型需要考虑完整的上下文信息

这种生成方式决定了：**每个新生成的token必须与之前所有生成的token进行注意力计算**。

## 2. 传统注意力计算的问题

在没有KV缓存的情况下，每次生成新token时的计算流程如下：

```
第1次生成：
- 输入：[token_0] （提示词）
- 计算：K_0, V_0 → Q_0 × K_0^T → 生成 token_1

第2次生成：
- 输入：[token_0, token_1]
- 计算：K_0, V_0, K_1, V_1 → Q_1 × [K_0; K_1]^T → 生成 token_2

第3次生成：
- 输入：[token_0, token_1, token_2]
- 计算：K_0, V_0, K_1, V_1, K_2, V_2 → Q_2 × [K_0; K_1; K_2]^T → 生成 token_3

...

第n次生成：
- 输入：[token_0, token_1, ..., token_{n-1}]
- 计算：K_0~K_{n-1}, V_0~V_{n-1} → Q_{n-1} × [K_0; ...; K_{n-1}]^T → 生成 token_n
```

**核心问题**：每次生成新token时，都需要**重新计算所有历史token的K和V向量**，存在大量重复计算！

## 3. KV缓存的解决方案

### 3.1 什么是KV缓存

KV缓存是一种**空间换时间**的优化技术，它将历史token的Key（K）和Value（V）向量保存起来，避免重复计算。

### 3.2 KV缓存的工作原理

使用KV缓存后的计算流程：

```
第1次生成：
- 输入：[token_0]
- 计算：K_0, V_0 → 保存到KV缓存 → Q_0 × K_0^T → 生成 token_1

第2次生成：
- 输入：[token_1]（只输入新token）
- 计算：K_1, V_1 → 追加到KV缓存 → Q_1 × [K_0（缓存）; K_1]^T → 生成 token_2

第3次生成：
- 输入：[token_2]
- 计算：K_2, V_2 → 追加到KV缓存 → Q_2 × [K_0（缓存）; K_1（缓存）; K_2]^T → 生成 token_3

...

第n次生成：
- 输入：[token_{n-1}]
- 计算：K_{n-1}, V_{n-1} → 追加到KV缓存 → Q_{n-1} × [K_0~K_{n-2}（缓存）; K_{n-1}]^T → 生成 token_n
```

### 3.3 为什么注意力计算必须使用KV缓存

当新生成的token与之前所有生成的token进行注意力计算时，**必须访问所有历史token的K和V向量**。而这些历史K和V向量：

1. **已经在之前的计算中生成过**
2. **不会再改变**（因为历史token已经确定）
3. **在每次新token生成时都需要用到**

因此，将这些历史K和V向量缓存起来是**最自然且最高效的选择**，可以避免大量重复计算。

## 4. FlashAttention中KV缓存的具体应用

### 4.1 有效序列长度计算

在FlashAttention的实现中，KV缓存的使用首先体现在**有效序列长度的计算**上：

```cpp
// 计算当前查询分块需要关注的有效KV序列长度
uint32_t noSkipKvS = kvSeqlen;
if (maskType != 0U) {  // 存在因果掩码时
    uint32_t diffS = kvSeqlen - qSeqlen;
    noSkipKvS = (qSBlockIdx + 1U) * curQSBlockTile + diffS;
    noSkipKvS = AscendC::Std::min((uint32_t)kvSeqlen, noSkipKvS);
}
```

这段代码确定了：对于当前新生成的token，需要从KV缓存中读取多少历史K和V向量来进行注意力计算。

### 4.2 KV序列分块处理

为了适应硬件内存限制，FlashAttention将KV缓存中的序列分成多个块处理：

```cpp
// 计算KV序列的总循环次数
uint32_t kvSLoopNumTotal = CeilDiv(noSkipKvS, MAX_KV_STACK_LEN);
```

这确保了即使生成很长的序列，也能高效地利用KV缓存进行注意力计算。

### 4.3 分页缓存管理

对于非常长的序列，FlashAttention支持**分页KV缓存**模式：

```cpp
// 分页缓存模式下的PV矩阵乘法
if constexpr (PAGED_CACHE_FLAG) {
    blockMmadPV(
        gP[gmOffsetP],  // P矩阵（注意力权重）
        gV[gmOffsetV],  // V矩阵（值张量，来自KV缓存）
        gOTmp[gmOffsetOTmp],  // 临时输出张量
        gBlockTable[blockBOffset],  // 块表（分页缓存管理）
        // 其他参数...
    );
}
```

分页缓存通过块表管理KV缓存的内存空间，提高了内存利用率，支持更长序列的生成。

## 5. KV缓存带来的优势

当新生成的token与所有历史token进行注意力计算时，KV缓存带来了以下关键优势：

### 5.1 性能提升

- **减少计算量**：避免重复计算历史token的K和V向量
- **降低内存访问**：减少全局内存访问次数，提高内存带宽利用率
- **加速推理过程**：时间复杂度从O(n²)接近O(n)

### 5.2 内存效率

- **空间复用**：KV缓存复用内存空间，避免每次生成新token时重新分配大量内存
- **灵活管理**：支持分页缓存模式，更灵活地管理内存空间
- **缓存优化**：结合FlashAttention的缓存层次优化，进一步提高效率

### 5.3 长序列支持

- **突破硬件限制**：允许生成更长的序列，突破单设备的内存限制
- **渐进式生成**：支持渐进式生成，每次只处理当前token的计算

## 6. 结论

当每个新生成的token需要与之前所有生成的token进行注意力计算时，**使用KV缓存是必然的选择**。这是因为：

1. 历史token的K和V向量不会改变，且在每次新token生成时都需要用到
2. 缓存这些向量可以避免大量重复计算，显著提高性能
3. KV缓存结合分块处理和分页管理，能够高效支持长序列生成

在FlashAttention等现代注意力实现中，KV缓存已经成为自回归推理优化的核心技术，对于提高模型的推理速度和降低部署成本具有重要意义。
